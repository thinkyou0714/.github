# Support

Need help with a thinkyou0714 project? Here is where to go:

## Routing order

1. Check README/docs in the affected repository.
2. Search existing issues and discussions.
3. Use Discussions for open-ended questions.
4. Use issue templates when you have a reproducible bug/feature request or Discussions are unavailable.

## Questions and discussion

- **GitHub Discussions** — per-repository Q&A (recommended for most questions)
- **Question issue template** — use when the repository has Discussions disabled or unsuitable
- **Search existing issues** — your question may already be answered

When opening a question issue, include target repository, environment, and what you already tried to reduce back-and-forth. Always open it in the affected repository.

## Bug reports

Open an issue using the **Bug report** template in the affected repository.

## Feature requests

Open an issue using the **Feature request** template, or start a Discussion to gauge interest first.

## Security issues

- For vulnerabilities, follow `SECURITY.md` and use private reporting first.
- If private reporting is unavailable, request a confidential channel first and keep public posts non-technical.
- Never post exploit details, proof-of-concept code, or sensitive indicators in public issues, discussions, or pull requests.
