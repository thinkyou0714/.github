## 0. Headline — Can Claude Code **on the web** use MCP / Skills / CLI today?

Claude Code web = an ephemeral **Ubuntu 24.04** container connected to a GitHub repo (NOT your
machine). It has **no access** to your local `~/.claude/`, local Obsidian REST API (`:27124`),
or local Ollama (`:4101`). What a repo commits is all a web session gets.

| Capability | Status today | Why | Fix |
|---|---|---|---|
| **CLI** | ⚠️ Partial | Node 20-22 / Python / git / jq / Docker / Postgres / Redis are pre-installed, so base CLI works. But **0/27** repos install their own deps → a web session must `npm ci` by hand. | Commit a POSIX **SessionStart hook** (`.claude/settings.json`) running `npm ci` idempotently. |
| **MCP** | ❌ 0/27 web-usable | Repo-root `.mcp.json` auto-loads on web, but **only HTTP/SSE remote servers work** (stdio/`npx`/`localhost` do NOT). The only repo with `.mcp.json` (`lab-infra`) is entirely localhost/Windows/stdio → breaks on web init. | Commit `.mcp.json` with HTTP/SSE servers only (e.g. GitHub-hosted MCP), never a committed token; keep localhost MCP local-only. |
| **SKILL** | ⚠️ 2/27 | Repo `.claude/skills/<name>/SKILL.md`, `.claude/commands/`, `.claude/rules/`, root `CLAUDE.md`/`AGENTS.md` all auto-load on web. Only **`tyl-monorepo` (14)** and **`fugu` (1)** commit skills; the other 25 give a web session nothing. | Commit 1-2 repo-relevant skills/commands + a root `AGENTS.md`/`CLAUDE.md`. |

**Root cause:** the whole `~/.claude` dev-OS is user-scoped (local-only). Nothing was ever
projected into individual repos, so a cloud session starts blank. This audit's 2026-07 pass fixes
that for the flagship public repos via a reusable **web-readiness template**
(see `docs/claude-code-web-readiness.md`).
